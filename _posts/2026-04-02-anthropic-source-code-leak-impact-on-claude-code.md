# Anthropic 源代碼洩漏：對 Claude Code 的意外打擊

Anthropic 的 Claude Code 源代碼洩漏事件引發了廣泛關注，這次洩漏源於包裝錯誤，並非安全漏洞。根據 Zscaler 的報告，Claude Code 的完整源代碼因未能正確排除源代碼映射文件（.map），意外在 GitHub 上公開。該事件可能使競爭對手獲得技術優勢，並對 Anthropic 的 IPO 計劃構成挑戰。

## 重點整理
- 洩漏事件因包裝錯誤導致完整源代碼公開。
- Anthropic 的市場競爭力可能因此削弱，影響其 3800 億美元 IPO 計劃。
- GitHub 在事件中為代碼托管平台，主要責任在於 Anthropic。
- 洩漏事件發生在 Claude Code 年收入達 25 億美元之際。
- 這是 Anthropic 在一年內的第二次洩漏事件。

## 技術細節
Claude Code 是一款由 Anthropic 開發的 AI 驅動編碼工具，洩漏事件的根源在於打包過程中未能正確排除源代碼映射文件（.map），導致完整源代碼意外公開。Claude Code 的 "Undercover Mode" 功能也被洩露，顯示其在公共開放原始碼項目中的隱秘貢獻。

## 產業與市場影響
事件發生的時機對 Anthropic 極為不利，Claude Code 正在企業市場取得成功，年收入達 25 億美元。洩漏可能使競爭對手在 AI 編碼工具市場中獲得技術優勢，並對 Anthropic 的 3800 億美元 IPO 計劃構成挑戰。目前尚無具體競爭對手的反應，但市場不確定性增加。

## 不同聲音
洩漏事件引發了對 Anthropic 操作安全性的質疑，特別是其自稱的「安全優先」定位。這是 Anthropic 在一年內第二次發生類似事件，顯示其內部安全措施可能存在問題。對於洩漏的長期市場影響，來源之間存在分歧，一些認為會嚴重影響市場地位，另一些則認為影響有限。

## 對台灣的意義
這次事件對台灣科技業提供了一個機會來審視自身的安全措施和風險管理策略。對於使用 Claude Code 的台灣開發者，需特別關注可能的安全漏洞和更新，以保障開發工作不受影響。
